# chordsketch for Kotlin/JVM

[ChordSketch](https://github.com/koedame/chordsketch) Kotlin/JVM bindings —
parse and render [ChordPro](https://www.chordpro.org/) and
[iReal Pro](https://www.irealpro.com/) chord charts from Kotlin or
Java via native Rust extensions generated by [UniFFI](https://mozilla.github.io/uniffi-rs/).

**Prebuilt native libraries are bundled in the JAR — no Rust toolchain required.**

## Installation

[![Maven Central](https://img.shields.io/maven-central/v/me.koeda/chordsketch)](https://central.sonatype.com/artifact/me.koeda/chordsketch)

Replace `VERSION` in the snippets below with the version shown in the badge above.

### Gradle (Kotlin DSL)

```kotlin
dependencies {
    implementation("me.koeda:chordsketch:VERSION")
}
```

### Gradle (Groovy DSL)

```groovy
dependencies {
    implementation 'me.koeda:chordsketch:VERSION'
}
```

### Maven

```xml
<dependency>
    <groupId>me.koeda</groupId>
    <artifactId>chordsketch</artifactId>
    <version>VERSION</version>
</dependency>
```

Requires **Java 17+** and **JNA** (pulled transitively).

## Quick Start

```kotlin
import uniffi.chordsketch.*

val source = """
{title: Amazing Grace}
{key: G}

[G]Amazing [G7]grace, how [C]sweet the [G]sound
""".trimIndent()

val html = parseAndRenderHtml(source, null, null)
val text = parseAndRenderText(source, null, null)
val pdf  = parseAndRenderPdf(source, null, null)  // ByteArray

println(version())
```

## API

### Rendering

All render functions accept three parameters:

| Parameter | Type | Description |
|-----------|------|-------------|
| `input` | `String` | ChordPro source text |
| `configJson` | `String?` | Preset name (`"guitar"`, `"ukulele"`) or inline RRJSON; `null` for defaults |
| `transpose` | `Byte?` | Semitone offset (-128..127); `null` defaults to 0 |

| Function | Returns | Throws | Description |
|----------|---------|--------|-------------|
| `parseAndRenderText(input, configJson, transpose)` | `String` | `ChordSketchException` | Plain text output |
| `parseAndRenderHtml(input, configJson, transpose)` | `String` | `ChordSketchException` | Full HTML document |
| `parseAndRenderPdf(input, configJson, transpose)` | `ByteArray` | `ChordSketchException` | Raw PDF bytes |

### iReal Pro conversion

| Function | Returns | Throws | Description |
|----------|---------|--------|-------------|
| `convertChordproToIrealb(input)` | `ConversionWithWarnings` | `ChordSketchException` | ChordPro → `irealb://` URL (lossy: drops lyrics, fonts, capo) |
| `convertIrealbToChordproText(input)` | `ConversionWithWarnings` | `ChordSketchException` | `irealb://` URL → rendered ChordPro text |
| `renderIrealSvg(input)` | `String` (SVG document) | `ChordSketchException` | `irealb://` URL → iReal Pro-style SVG chart |
| `renderIrealPng(input)` | `ByteArray` (PNG bytes) | `ChordSketchException` | `irealb://` URL → PNG image (300 DPI default, A4-equivalent canvas) |
| `renderIrealPdf(input)` | `ByteArray` (PDF bytes) | `ChordSketchException` | `irealb://` URL → single-page A4 PDF document |
| `parseIrealb(input)` | `String` (JSON) | `ChordSketchException` | `irealb://` URL → AST-shaped JSON mirroring `IrealSong` |
| `serializeIrealb(input)` | `String` (URL) | `ChordSketchException` | AST-shaped JSON → `irealb://` URL (round-trips with `parseIrealb`) |

`ConversionWithWarnings` exposes `output: String` and
`warnings: List<String>`. Each warning is a
`"<kind>: <message>"` string (`kind` is `lossy-drop`,
`approximated`, or `unsupported`). The `output` of
`convertIrealbToChordproText` is the rendered text
representation of the converted song, not raw ChordPro source.

```kotlin
val result = convertChordproToIrealb("{title: Test}\n[C]Hello")
println(result.output)   // "irealb://..."
println(result.warnings) // [lossy-drop: lyrics are dropped, ...]
```

> **API naming:** functions use camelCase as generated by UniFFI from the Rust
> source. Import from `uniffi.chordsketch`.

### Validation

```kotlin
val errors: List<ValidationError> = validate(source)  // empty if clean
errors.forEach { println("line ${it.line}, column ${it.column}: ${it.message}") }
```

### Utility

```kotlin
val v: String = version()  // e.g. "0.2.0"
```

## Options

```kotlin
// Transpose up 2 semitones with the ukulele preset
val html = parseAndRenderHtml(source, "ukulele", 2)

// Inline RRJSON configuration
val html = parseAndRenderHtml(
    source,
    """{"settings": {"notation": "solfege"}}""",
    null,
)
```

## Error Handling

Functions throw `ChordSketchException` on invalid configuration:

```kotlin
try {
    val html = parseAndRenderHtml(source, "{ bad json !!!", null)
} catch (e: ChordSketchException.InvalidConfig) {
    println("Bad config: ${e.reason}")
} catch (e: ChordSketchException.NoSongsFound) {
    println("No songs in input")
}
```

`ChordSketchException` subtypes:
- `NoSongsFound` — the input produced no parseable songs
- `InvalidConfig(reason: String)` — invalid preset name or RRJSON
- `ConversionFailed(reason: String)` — ChordPro ↔ iReal conversion rejected the input (#2067)

Parse errors in the ChordPro input are **not** thrown — the renderer is lenient
and produces a best-effort result. Call `validate()` to surface diagnostics.
