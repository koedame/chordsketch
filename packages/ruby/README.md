# chordsketch

[ChordSketch](https://github.com/koedame/chordsketch) Ruby gem —
parse and render [ChordPro](https://www.chordpro.org/) files from Ruby via
native Rust extensions generated by [UniFFI](https://mozilla.github.io/uniffi-rs/).

**Native gem — prebuilt for each platform, no Rust toolchain required.**

## Installation

```bash
gem install chordsketch
```

Or add to your `Gemfile`:

```ruby
gem "chordsketch"
```

Requires Ruby 3.0+.

Prebuilt native libraries are bundled for five platforms:

| Platform | Architecture |
|----------|-------------|
| Linux | x86\_64, arm64 |
| macOS | x86\_64, arm64 (Apple Silicon) |
| Windows | x86\_64 |

## Quick Start

```ruby
require "chordsketch"

source = <<~CHORDPRO
  {title: Amazing Grace}
  {key: G}

  [G]Amazing [G7]grace, how [C]sweet the [G]sound
CHORDPRO

html = Chordsketch.parse_and_render_html(source, nil, nil)
text = Chordsketch.parse_and_render_text(source, nil, nil)
pdf  = Chordsketch.parse_and_render_pdf(source, nil, nil)  # String (binary)

puts Chordsketch.version
```

> **Module name:** `Chordsketch` (capital C, rest lowercase). This is the
> namespace generated by UniFFI.

## API

### Rendering

All render methods accept three arguments:

| Parameter | Type | Description |
|-----------|------|-------------|
| `input` | `String` | ChordPro source text |
| `config_json` | `String \| nil` | Preset name (`"guitar"`, `"ukulele"`) or inline RRJSON; `nil` for defaults |
| `transpose` | `Integer \| nil` | Semitone offset (-128..127); `nil` defaults to 0 |

| Method | Returns | Description |
|--------|---------|-------------|
| `Chordsketch.parse_and_render_text(input, config_json, transpose)` | `String` | Plain text output |
| `Chordsketch.parse_and_render_html(input, config_json, transpose)` | `String` | Full HTML document |
| `Chordsketch.parse_and_render_pdf(input, config_json, transpose)` | `String` | Raw PDF bytes (binary string) |

### Validation

```ruby
errors = Chordsketch.validate(source)  # Array<String> — empty if clean
errors.each { |msg| puts msg }
```

### Utility

```ruby
puts Chordsketch.version  # e.g. "0.2.0"
```

## Options

```ruby
# Transpose up 2 semitones with the ukulele preset
html = Chordsketch.parse_and_render_html(source, "ukulele", 2)

# Inline RRJSON configuration
html = Chordsketch.parse_and_render_html(
  source,
  '{"settings": {"notation": "solfege"}}',
  nil,
)
```

## Error handling

Methods raise `Chordsketch::ChordSketchError` on invalid configuration:

```ruby
begin
  Chordsketch.parse_and_render_html(source, "{ bad json !!!", nil)
rescue Chordsketch::ChordSketchError => e
  puts e.message
end
```

Parse errors in the ChordPro input are **not** raised — the renderer is lenient
and produces a best-effort result. Call `validate` to surface diagnostics.
