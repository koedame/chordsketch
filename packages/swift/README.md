# ChordSketch for Swift

[ChordSketch](https://github.com/koedame/chordsketch) Swift package —
parse and render [ChordPro](https://www.chordpro.org/) files on Apple platforms
via a prebuilt XCFramework backed by Rust.

**Prebuilt XCFramework — no Rust build step required.**
Supports macOS 12+ and iOS 15+.

## Installation

[![GitHub Release](https://img.shields.io/github/v/release/koedame/chordsketch)](https://github.com/koedame/chordsketch/releases/latest)

Replace the version string in the snippet below with the
[latest release tag](https://github.com/koedame/chordsketch/releases/latest)
shown in the badge above.

Add the package to your `Package.swift`:

```swift
// swift-tools-version: 5.9
import PackageDescription

let package = Package(
    // ...
    dependencies: [
        .package(
            url: "https://github.com/koedame/chordsketch",
            from: "0.2.0"  // replace with the latest release tag
        ),
    ],
    targets: [
        .target(
            name: "YourTarget",
            dependencies: [
                .product(name: "ChordSketch", package: "chordsketch")
            ]
        ),
    ]
)
```

Or add via Xcode: **File → Add Package Dependencies**, enter
`https://github.com/koedame/chordsketch`.

### CocoaPods

[![CocoaPods](https://img.shields.io/cocoapods/v/ChordSketch)](https://cocoapods.org/pods/ChordSketch)

Add to your `Podfile`:

```ruby
pod 'ChordSketch'
```

Then run:

```bash
pod install
```

## Quick Start

```swift
import ChordSketch

let source = """
{title: Amazing Grace}
{key: G}

[G]Amazing [G7]grace, how [C]sweet the [G]sound
"""

let html = try parseAndRenderHtml(input: source, configJson: nil, transpose: nil)
let text = try parseAndRenderText(input: source, configJson: nil, transpose: nil)
let pdf  = try parseAndRenderPdf(input: source, configJson: nil, transpose: nil)  // [UInt8]

print(version())
```

## API

### Rendering

All render functions accept three parameters:

| Parameter | Type | Description |
|-----------|------|-------------|
| `input` | `String` | ChordPro source text |
| `configJson` | `String?` | Preset name (`"guitar"`, `"ukulele"`) or inline RRJSON; `nil` for defaults |
| `transpose` | `Int8?` | Semitone offset (-128..127); `nil` defaults to 0 |

| Function | Returns | Throws | Description |
|----------|---------|--------|-------------|
| `parseAndRenderText(input:configJson:transpose:)` | `String` | `ChordSketchError` | Plain text output |
| `parseAndRenderHtml(input:configJson:transpose:)` | `String` | `ChordSketchError` | Full HTML document |
| `parseAndRenderPdf(input:configJson:transpose:)` | `[UInt8]` | `ChordSketchError` | Raw PDF bytes |

### iReal Pro conversion

| Function | Returns | Throws | Description |
|----------|---------|--------|-------------|
| `convertChordproToIrealb(input:)` | `ConversionWithWarnings` | `ChordSketchError` | ChordPro → `irealb://` URL (lossy: drops lyrics, fonts, capo) |
| `convertIrealbToChordproText(input:)` | `ConversionWithWarnings` | `ChordSketchError` | `irealb://` URL → rendered ChordPro text |
| `renderIrealSvg(input:)` | `String` (SVG document) | `ChordSketchError` | `irealb://` URL → iReal Pro-style SVG chart |
| `renderIrealPng(input:)` | `[UInt8]` (PNG bytes) | `ChordSketchError` | `irealb://` URL → PNG image (300 DPI default, A4-equivalent canvas) |
| `renderIrealPdf(input:)` | `[UInt8]` (PDF bytes) | `ChordSketchError` | `irealb://` URL → single-page A4 PDF document |
| `parseIrealb(input:)` | `String` (JSON) | `ChordSketchError` | `irealb://` URL → AST-shaped JSON mirroring `IrealSong` |
| `serializeIrealb(input:)` | `String` (URL) | `ChordSketchError` | AST-shaped JSON → `irealb://` URL (round-trips with `parseIrealb`) |

`ConversionWithWarnings` exposes `output: String` and
`warnings: [String]`. Each warning is a `"<kind>: <message>"`
string (`kind` is `lossy-drop`, `approximated`, or
`unsupported`). The `output` of `convertIrealbToChordproText`
is the rendered text representation of the converted song,
not raw ChordPro source.

```swift
let result = try convertChordproToIrealb(input: "{title: Test}\n[C]Hello")
print(result.output)    // "irealb://..."
print(result.warnings)  // ["lossy-drop: lyrics are dropped", ...]
```

> **API naming:** functions use camelCase as generated by UniFFI from the Rust
> source. The Swift package name is `ChordSketch` (PascalCase for the import).

### Validation

```swift
let errors: [ValidationError] = validate(input: source)  // empty if clean
for e in errors {
    print("line \(e.line), column \(e.column): \(e.message)")
}
```

```swift
let v: String = version()  // e.g. "0.2.0"
```

## Options

```swift
// Transpose up 2 semitones with the ukulele preset
let html = try parseAndRenderHtml(input: source, configJson: "ukulele", transpose: 2)

// Inline RRJSON configuration
let html = try parseAndRenderHtml(
    input: source,
    configJson: #"{"settings": {"notation": "solfege"}}"#,
    transpose: nil
)
```

## Error Handling

Functions throw `ChordSketchError` on invalid configuration:

```swift
do {
    let html = try parseAndRenderHtml(input: source, configJson: "{ bad json !!!", transpose: nil)
} catch ChordSketchError.invalidConfig(let reason) {
    print("Bad config: \(reason)")
} catch ChordSketchError.noSongsFound {
    print("No songs in input")
}
```

`ChordSketchError` cases:
- `.noSongsFound` — the input produced no parseable songs
- `.invalidConfig(reason: String)` — invalid preset name or RRJSON
- `.conversionFailed(reason: String)` — ChordPro ↔ iReal conversion rejected the input (#2067)

Parse errors in the ChordPro input are **not** thrown — the renderer is lenient
and produces a best-effort result. Call `validate` to surface diagnostics.
